# Phase2-B Output/Error-Process Decoder Design

## Current Step

`current_step`: step 1-6 of the long research loop.

This design is a conditional fallback for Phase2. It should become active only
if Phase2-R.1 `PatchEncoderFutureStateAlignmentConfWeighted` fails or reaches
only a partial pass. It is documented now because existing artifacts already

## Step 1: Research Analysis

[Fact] Phase1-A.6 already tested a simple output-process idea:
`PatchEncoderTrajectoryBasisResidual`.

That candidate kept the fixed-head base trajectory and added a low-rank,
position-aware residual:

$$
\hat{Y}_{1:H}
=
\hat{Y}^{base}_{1:H}
+
g_H \odot B_\phi(p_{1:H}) A_\theta(Z).
$$

Its remote gate did not pass:

| Comparison | MSE wins | Mean relative MSE |
| --- | ---: | ---: |
| vs `PatchEncoderFixedHead` | `5/12` | `+0.67%` |
| vs `PatchEncoderFixedHeadAdapter` | `6/12` | `+0.49%` |
| vs `PatchEncoderStepSpecificStateAdapter` | `4/12` | `+0.33%` |

[Inference] The negative result is useful. It says that the next
output-process candidate should not be another static, low-rank position basis
attached to a horizon-specific fixed head.

[Fact] The later target-set carrier R.3 changed the setting. It made a
one-model decoder possible and exposed target-side states:

$$
U_j=D_\theta(Q_j,Z),
$$

but the H720 step-wise diagnosis shows a remaining output/error process:

| Dataset | Segment | R.3 vs FixedHead | Phase2-A vs R.3 | Interpretation |
| --- | --- | ---: | ---: | --- |
| `ETTh2` | `1-96` | `-1.80%` | `+6.42%` | R.3 early gain erased by Phase2-A |
| `ETTh2` | `193-336` | `+4.07%` | `+2.54%` | both stages worsen middle region |
| `ETTm1` | `337-720` | `+3.03%` | `-1.73%` | R.3 late weak region repaired by Phase2-A |
| `Weather` | `1-96` | `-4.04%` | `+0.52%` | R.3 early gain slightly erased |

[Strong Evidence] The issue is step-region structured. It is not visible from
only mean MSE/MAE. The relative step curves have many sign changes and distinct
early/middle/late behavior.

## Step 2: Problem

The current target-set decoder predicts each target segment by applying a
target-state-conditioned FiLM readout to a dense history representation:

$$
\hat{Y}_{a_j:b_j}
=
O_\theta(r\odot(1+\gamma(U_j))+\beta(U_j)).
$$

This treats each segment's point values as the direct output of a target state.
It does not explicitly model the prediction trajectory as an output process:

$$
\hat{Y}_{1:H}
=
\{\hat{y}_1,\hat{y}_2,\dots,\hat{y}_H\},
$$

where forecast errors are correlated across nearby future steps and grow or
change regime across horizon regions.

The problem is therefore:

> How should a one-model target-set decoder model the future output/error
> process so that it can preserve early-step gains, correct middle/late weak
> regions, and avoid dataset-specific step-region conflict?

## Step 3: Is The Problem Worth Studying?

[Strong Evidence] Yes, but only under a narrow definition.

The problem is real because:

1. H720 step-wise curves show region-specific sign changes; average MSE hides
   this structure.
2. R.3 improves some early regions while worsening middle/late regions.
3. Phase2-A can repair `ETTm1` late error while breaking `ETTh2` broadly,
   implying that target states influence output process but do not control it
   reliably.
4. QDF argues that standard MSE treats future steps as independent and
   identity-weighted, which matches the observed step dependency issue.

The problem is worth studying only if the design is not a repeat of A.6:

- not a static position-only residual basis;
- not a stronger fixed-head patch;
- not a future teacher alignment loss without output-process control;
- not MoE before a stable residual/error process state exists.

## Step 4: Idea

Candidate family: `Target-Conditioned Error-Process Decoder`.

Candidate implementation name, if activated:
`PatchEncoderErrorProcessDecoder`.

The model should decompose prediction into:

$$
\hat{Y}_{1:H}
=
\hat{Y}^{base}_{1:H}
+
\Delta_\theta(X,T),
$$

but unlike A.6, the residual process is not generated by static position bases.
It is generated by a causal or structured process over target-set states:

$$
E_j = G_\theta(U_j, q_j, c_{j-1}),
$$

$$
c_j = R_\theta(c_{j-1}, E_j),
$$

$$
\Delta Y_{a_j:b_j}=O^e_\theta(E_j,c_j).
$$

Here:

- $U_j$ is the R.3 target-side state;
- $q_j$ is the target segment feature;
- $c_j$ is a compact error-process state passed across future segments;
- $\Delta Y$ is an additive residual over the target-set prediction.

This is a residual decoder, but its residual is target-conditioned and
stateful. The key claim is:

> The residual should be a future-step process conditioned on target-set states,
> not a fixed low-rank function of position.

## Step 5: Theory Check

### Why This Is Different From A.6

A.6 used:

$$
\Delta Y_{t,c}=\sum_k B_{t,k}A_{c,k}.
$$

The basis $B_{t,k}$ depended on future position, and the coefficients depended
on history. It lacked feedback from target-state geometry and did not condition
residual evolution on earlier future regions.

Phase2-B instead uses:

$$
\Delta Y_{a_j:b_j}
=
f_\theta(U_j,q_j,c_{j-1}),
$$

so the residual process can respond to the same target-state carrier that
drives the main decoder.

### Why This Is Different From Phase2-A

Phase2-A aligned $U_j$ to a training-only future teacher. It did not explicitly
constrain how forecast errors evolve over steps.

Phase2-B directly models output residuals:

$$
R_{1:H}=Y_{1:H}-\hat{Y}^{base}_{1:H}.
$$

It asks whether the decoder should learn the residual process rather than a
latent future-state proxy.

### Objective

The first version should keep the prediction objective simple:

$$
\mathcal{L}_{pred}=\operatorname{MSE}(\hat{Y},Y).
$$

It should add only small regularizers that are directly tied to the error
process:

$$
\mathcal{L}_{energy}
=
\lambda_e
\frac{\|\Delta Y\|_2^2}{\|\hat{Y}^{base}\|_2^2+\epsilon},
$$

$$
\mathcal{L}_{smooth}
=
\lambda_s \|\nabla_t^2 \Delta Y\|_2^2.
$$

QDF-style covariance should initially be diagnostic, not the first training
loss. If the process decoder works, a later ablation can replace or supplement
MSE with a learned step covariance objective.

- `PatchEncoderPrefixRiskWeighted` R.3;
- Phase1-A.6 `PatchEncoderTrajectoryBasisResidual`, used as a failed

### Matrix

Use the same Phase2 matrix:

- target horizons: `{96,192,336,720}`;
- seed: `2021`;
- epochs: `100`;
- local smoke first, remote gate only after commit/push.

### Required Artifacts

The gate must write:

- `metrics_by_target_horizon.csv`;
- `metrics_by_horizon.csv`;
- `metrics_by_segment.csv`;
- `prefix_consistency.csv`;
- `error_process_stats.csv`;
- `effective_config.json`;
- `training_log.csv`.

`error_process_stats.csv` should define:

- residual/base MAE ratio;
- residual energy by segment;
- residual second-difference smoothness;
- error-process state norm by segment;
- segment-to-segment state cosine;
- H720 early/middle/late relative MSE summary.

### Implementation Interface Contract

Phase2-B should reuse the existing `baselines/patch_encoder_target_set_decoder`
data path and tensor contracts unless the R.1 result forces a base-architecture
change. The first implementation should be a new candidate name, not a silent
mutation of R.3:

- model class: either a subclass/wrapper around `PatchEncoderTargetSetDecoder`,
  or a local sibling with the same public `forward(x, pred_len, ...)` contract;
- trainer: start from `baselines/patch_encoder_target_set_decoder/train.py`
  to preserve mixed-horizon sampling, prefix consistency evaluation, and
  artifact layout;
- run name: `PatchEncoderErrorProcessDecoder`;
- output root for local smoke:
  `artifacts/runs/smoke_phase2_error_process_decoder`;
- output root for remote gate:
  `/home/yingch/exp_outputs/r-2026-fatst/phase2_error_process_decoder`.

Forward contract:

```text
forward(x, pred_len, return_components=False)
```

must return a prediction tensor:

$$
\hat{Y}\in\mathbb{R}^{B\times H\times C}.
$$

When `return_components=True`, it must additionally return:

- `prediction`: final denormalized prediction, shape `[B,H,C]`;
- `base_prediction`: R.3 base prediction before error-process residual,
  shape `[B,H,C]`;
- `error_residual`: additive residual after any RevIN denormalization needed
  for comparability, shape `[B,H,C]`;
- `error_residual_norm`: residual in normalized model space, shape `[B,H,C]`;
- `error_process_states`: compact process states, shape `[B,C,J,d_e]` or
  `[B,J,d_e]` if channel-shared;
- `target_states`: inherited R.3 target states, shape `[B,C,J,d]`;
- `prefix_residual_norm`: inherited R.3 prefix residual if active.

RevIN boundary:

- the base R.3 segment values are produced in normalized space;
- the error residual should first be produced in normalized space and added
  before RevIN denormalization;
- the reported `error_residual` should be transformed to data scale only for
  diagnostics.

Prefix consistency invariant:

- for any short horizon $h$ and long horizon $H=720$, the first $h$ output
  steps must be generated by the same target states and the same error-process
  recurrence prefix;
- the implementation must not use requested final horizon $H$ to change already
  emitted residual states except through the set of available future target
  queries;
- `prefix_consistency.csv` remains a required artifact and must stay at
  numerical-zero level for a pass.

Initialization:

- error residual head final projection should be zero-initialized, or an
  explicit scalar gate should be initialized close to zero;
- the first epoch should therefore behave close to R.3, not destroy its early
  gains.

Minimal default hyperparameters:

- `error_process_dim=64`;
- `error_process_layers=1`;
- `error_residual_gate_init=-4.0`;
- `error_energy_weight=1e-4`;
- `error_smoothness_weight=1e-4`;
- `segment_len=48` inherited from R.3.

### Pass Conditions

The candidate should not be accepted only because one dataset improves.

Minimum pass:

1. vs R.3 mean relative MSE < `0`;
2. vs R.3 at least `7/12` MSE wins;
3. no dataset mean degradation > `+0.3%` vs R.3;
4. H720 step-region diagnostics improve at least two of:
   - `ETTh2 193-336`;
   - `ETTh2 337-720`;
   - `ETTm1 337-720`;
   - `Weather 1-96`;
5. residual/base ratio is nonzero but controlled, not an uncontrolled large
   correction;
6. prefix mismatch remains numerical-zero level.

Paper-core pass additionally requires:

- improvement is explainable through error-process diagnostics, not only final
  aggregate MSE;
- same-parameter dense residual control does not explain the gain;
- the mechanism supports a clear story:
  target-set forecasting needs a structured output/error process, not only
  stronger target states.

### Rollback

If Phase2-B fails:

- do not continue adding residual capacity;
- do not add MoE on residual states;
- return to loop step 2-3 and consider objective-level approaches such as
  QDF-style step covariance weighting, or leave decoder innovation and move to
  a different paper contribution.
